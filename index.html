AMD / ARM 通用 | DD失败 · 失联救援
2026-03-31 · 基于新版控制台界面撰写

在折腾甲骨文云（Oracle Cloud）时，使用一键DD脚本导致系统崩溃、SSH无法连接（俗称“变砖”）是常见问题。由于官方控制台频繁更新，旧教程大多失效。本教程针对最新版界面，提供两种最可靠的救援方案，让失联实例原地复活。

核心原理：Oracle Cloud 的引导卷是可分离的虚拟硬盘，通过替换引导卷或挂载到其他实例重灌系统，即可恢复。

方法一 · 官方替换引导卷

无需额外实例 | 官方原生

原理：利用Oracle云原生功能，直接替换引导卷镜像，元数据注入root密码/公钥，快速恢复纯净系统。

操作步骤

  • 进入失联实例详情页 → 顶部【更多操作】或下滑至【引导卷】区域 → 点击 “替换引导卷”。
  • 填写映像OCID：点击实例现有的引导卷 → 进入关联的“映像”详情 → 复制OCID（格式如 ocid1.image.oc1...）→ 粘贴到“输入映像OCID”框内。
  • 保留引导卷？ 一般不勾选（除非想保留当前损坏的系统作备份）。
  • 高级选项（必看！） 展开【高级选项】→ 【元数据】→ 名称填写 user_data，值填写下方脚本的 Base64编码（用于自动设置root密码/开启SSH）。
注入脚本（Base64加密前的原始内容）：
#!/bin/bash
echo root:你的新密码 | chpasswd root
sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin yes/g' /etc/ssh/sshd_config
sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication yes/g' /etc/ssh/sshd_config
rm -rf /etc/ssh/sshd_config.d/* && rm -rf /etc/ssh/ssh_config.d/*
/etc/init.d/ssh* restart

🔐 记得将“你的新密码”改为复杂密码，然后用任意在线Base64工具编码后填入user_data值。

点击【保存】后，引导卷会经历“分离 → 附加”状态变更，等待3-5分钟，实例自动启动，即可用新密码SSH登录。

免费账户限制：所有引导卷总容量不得超过200GB，若已开满4台AMD（每台47G）则不适用此方法，请使用方法二。

方法二 · 分离硬盘 + 挂载DD救援

万能方案 | 无容量限制

将“变砖”实例的引导卷拆下，挂载到另一台正常实例上，写入新系统镜像后再装回。该方式不受200GB限制，也支持跨发行版。

详细流程

① 准备救援实例：同账号内必须有一台运行中的Linux实例（临时新建免费实例即可，架构最好与失联实例相同）。

② 停止失联实例（注意是“停止”，不是终止），然后进入【引导卷】→ 点击右侧菜单 → 分离。

③ 挂载到救援实例：导航至【存储】→【块存储卷】→ 找到刚才分离的引导卷（类型显示“引导卷”）→ 点击【附加到实例】→ 选择救援实例，附件类型选 半虚拟化 → 确认。

④ SSH登录救援实例，识别设备名：执行 lsblk 查看新挂载的磁盘，通常为 /dev/sdb（ARM也可能为/dev/sdc或nvme设备）。务必确认大小（一般为50G），避免写错系统盘。

下载并DD官方纯净系统（推荐）

根据你的失联实例架构选择对应命令（可在救援实例后台执行，建议使用screen防止断开）:

🔵 AMD (x86_64) 救砖 - Debian 12
curl -Lo- "https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-nocloud-amd64.tar.xz" | tar -xJO | dd of=/dev/sdb bs=1M
🟢 ARM (arm64) 救砖 - Debian 12
curl -Lo- "https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-nocloud-arm64.tar.xz" | tar -xJO | dd of=/dev/sdb bs=1M
DD过程大约20~30分钟。如果设备名不是/dev/sdb，请替换为实际名称（如/dev/sdc）。

快速DD包（第三方救援包，体积小速度更快）

适用于ARM架构，救援包较小（约3~10分钟恢复）：

wget --no-check-certificate https://github.com/MIKU-N/OracleBackup/raw/main/dabian.sdb.gz?download -O debian10.img.gz
gzip -dc debian10.img.gz | dd of=/dev/sdb

默认用户名root 密码 10086.fit 。使用后建议立即修改密码。

⑤ 收尾工作

等待DD执行完毕（可通过watch -n 5 pkill -USR1 ^dd$ 监控进度）。完成后回到控制台：

  • 【存储】→ 【块存储卷】→ 将该引导卷从救援实例分离。
  • 回到原失联实例详情页 → 【引导卷】→ 【附加块存储卷】→ 选择刚刚处理过的引导卷重新挂载。
  • 启动原实例，尝试SSH连接。
若仍无法SSH： 使用“控制台连接” → “启动Cloud Shell连接” 进入实例，运行 passwd root 设置密码，编辑 /etc/ssh/sshd_config 开启PermitRootLogin和PasswordAuthentication，执行 systemctl restart sshd 即可。

高阶技巧 & 常见问题

如何自制DD镜像包？

将你想备份的ARM/AMD引导卷挂载到一台配置较高的备用实例上，执行：

dd if=/dev/sdb | gzip > /root/backup.img.gz

恢复时使用 gzip -dc backup.img.gz | dd of=/dev/sdb 即可，方便日后批量部署。

DD后卡Grub / 无法引导怎么办？

可尝试在救援实例中格式化引导卷分区表再DD（先dd if=/dev/zero of=/dev/sdb bs=1M count=100 清引导区），然后重新写入系统镜像。或者使用 bash InstallNET.sh 脚本修复引导。

wget --no-check-certificate -qO InstallNET.sh 'https://raw.githubusercontent.com/leitbogioro/Tools/master/Linux_reinstall/InstallNET.sh' && chmod a+x InstallNET.sh
bash InstallNET.sh -debian 12 -pwd '你的新密码'

免费账号超过200G怎么办？

务必使用【方法二】挂载救援模式，删除失联引导卷内容并重灌系统。也可以考虑升级付费账户或清理其他实例的多余引导卷。
