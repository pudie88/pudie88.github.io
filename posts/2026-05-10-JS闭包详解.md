# JS 闭包详解：从入门到应用

> 标签：`JavaScript`  
> 日期：2026-05-10

## 什么是闭包

闭包（Closure）是指**函数可以记住并访问其词法作用域，即使函数在词法作用域之外执行**。

简单说：一个函数"记住"了它被定义时的环境。

```javascript
function makeCounter() {
  let count = 0           // count 在 makeCounter 的作用域里

  return function () {
    count++               // 内层函数可以访问外层的 count
    return count
  }
}

const counter = makeCounter()
console.log(counter())   // 1
console.log(counter())   // 2
console.log(counter())   // 3
```

`counter` 函数执行完之后，`count` 没有被销毁，因为内层函数持有对它的引用——这就是闭包。

## 闭包是怎么形成的

JavaScript 采用**词法作用域**（静态作用域）：函数的作用域在定义时就确定了，而不是调用时。

每次函数调用都会创建一个新的**执行上下文**，包含一个**变量环境**。当内层函数引用外层变量时，JS 引擎不会销毁外层的变量环境，而是把它保留在内存中，这就形成了闭包。

```javascript
function outer() {
  const msg = 'hello'        // 外层变量

  function inner() {
    console.log(msg)         // inner 闭包了 outer 的 msg
  }

  return inner
}

const fn = outer()
fn()  // 'hello' —— outer 已执行完，但 msg 还在
```

## 常见应用场景

### 1. 数据私有化（模块模式）

```javascript
function createUser(name) {
  let _loginCount = 0          // 私有变量，外部无法直接访问

  return {
    getName: () => name,
    login() {
      _loginCount++
      console.log(`${name} 第 ${_loginCount} 次登录`)
    },
    getLoginCount: () => _loginCount
  }
}

const user = createUser('Alice')
user.login()         // Alice 第 1 次登录
user.login()         // Alice 第 2 次登录
console.log(user.getLoginCount())  // 2
// user._loginCount  // undefined，无法访问
```

### 2. 函数工厂

```javascript
function multiply(x) {
  return (y) => x * y    // 记住了 x
}

const double = multiply(2)
const triple = multiply(3)

console.log(double(5))   // 10
console.log(triple(5))   // 15
```

### 3. 防抖函数（实际项目常用）

```javascript
function debounce(fn, delay) {
  let timer = null          // timer 被闭包持有

  return function (...args) {
    clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
    }, delay)
  }
}

const onSearch = debounce((keyword) => {
  console.log('搜索：', keyword)
}, 300)

// 快速连续触发，只有最后一次会执行
onSearch('v')
onSearch('vu')
onSearch('vue')   // 300ms 后执行：搜索：vue
```

## 经典陷阱：循环中的闭包

```javascript
// 错误写法 —— 输出 5 5 5 5 5
for (var i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 100)
}
```

原因：`var` 声明的 `i` 是全局的，循环结束时 `i` 已经是 5，所有回调共享同一个 `i`。

**修复方式一：用 `let`（推荐）**

```javascript
for (let i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 100)  // 0 1 2 3 4
}
```

`let` 是块级作用域，每次循环都是一个新的 `i`。

**修复方式二：IIFE 手动创建作用域**

```javascript
for (var i = 0; i < 5; i++) {
  ;(function (j) {
    setTimeout(() => console.log(j), 100)
  })(i)
}
```

## 注意内存泄漏

闭包会延长变量的生命周期。如果不注意，可能造成内存泄漏：

```javascript
function bindEvent() {
  const el = document.getElementById('btn')
  const bigData = new Array(100000).fill('*')  // 大对象

  el.addEventListener('click', () => {
    console.log(bigData.length)   // bigData 被闭包持有，无法回收
  })
}
```

改进：用完就释放引用，或者把不必要的变量移出闭包范围。

## 总结

| 特性 | 说明 |
|------|------|
| 核心原理 | 内层函数持有外层作用域的引用 |
| 常见用途 | 数据私有化、函数工厂、节流防抖 |
| 注意事项 | 循环陷阱、避免不必要的内存占用 |

闭包是 JavaScript 的核心特性之一，理解它能帮你写出更优雅的函数式代码。
