# Vue3 响应式原理深度解析

> 标签：`Vue` `原理`  
> 日期：2026-05-15

## Vue3 响应式 vs Vue2

| | Vue2 | Vue3 |
|---|---|---|
| 实现方式 | `Object.defineProperty` | `Proxy` |
| 数组检测 | 需要特殊处理 | 原生支持 |
| 新增属性 | 需要 `Vue.set` | 自动检测 |
| 性能 | 初始化时递归遍历 | 懒代理，按需收集 |

## Proxy 基础

Vue3 响应式的核心是 ES6 的 `Proxy`，它可以拦截对象的任何操作：

```javascript
const raw = { name: 'Vue', version: 3 }

const proxy = new Proxy(raw, {
  get(target, key) {
    console.log(`读取了 ${key}`)
    return Reflect.get(target, key)
  },
  set(target, key, value) {
    console.log(`设置了 ${key} = ${value}`)
    return Reflect.set(target, key, value)
  }
})

proxy.name        // 读取了 name
proxy.version = 4 // 设置了 version = 4
```

## reactive 的实现原理

`reactive()` 本质上就是对对象做 Proxy 代理，并在 `get` 时**收集依赖**，在 `set` 时**触发更新**。

```javascript
// 简化版实现，帮助理解原理
function reactive(obj) {
  return new Proxy(obj, {
    get(target, key, receiver) {
      track(target, key)          // 收集依赖
      const res = Reflect.get(target, key, receiver)
      // 如果值还是对象，继续代理（懒代理）
      if (typeof res === 'object' && res !== null) {
        return reactive(res)
      }
      return res
    },
    set(target, key, value, receiver) {
      const res = Reflect.set(target, key, value, receiver)
      trigger(target, key)        // 触发更新
      return res
    }
  })
}
```

### 依赖收集（track）

每次读取响应式数据时，记录"谁在用这个数据"：

```
targetMap = WeakMap {
  obj → depsMap = Map {
    'key' → dep = Set { effect1, effect2, ... }
  }
}
```

```javascript
let activeEffect = null

function track(target, key) {
  if (!activeEffect) return      // 没有正在运行的 effect，不需要收集

  let depsMap = targetMap.get(target)
  if (!depsMap) targetMap.set(target, (depsMap = new Map()))

  let dep = depsMap.get(key)
  if (!dep) depsMap.set(key, (dep = new Set()))

  dep.add(activeEffect)          // 把当前 effect 加入依赖集合
}
```

### 触发更新（trigger）

数据变化时，找出所有依赖这个数据的 effect 并重新执行：

```javascript
function trigger(target, key) {
  const depsMap = targetMap.get(target)
  if (!depsMap) return

  const dep = depsMap.get(key)
  dep && dep.forEach(effect => effect())
}
```

## ref 的原理

`ref` 用来包装基本类型值（数字、字符串等）。因为 Proxy 只能代理对象，所以 `ref` 把值包装成 `{ value: xxx }` 的对象形式：

```javascript
function ref(value) {
  return {
    get value() {
      track(this, 'value')    // 读取时收集依赖
      return value
    },
    set value(newVal) {
      value = newVal
      trigger(this, 'value')  // 设置时触发更新
    }
  }
}
```

这就是为什么在 JS 中访问 ref 要加 `.value`，而在模板里不需要——模板编译时会自动处理。

## computed 的原理

`computed` 本质是一个**懒执行、带缓存**的 effect：

```javascript
function computed(getter) {
  let dirty = true     // 标记是否需要重新计算
  let value

  const effect = createEffect(getter, {
    lazy: true,
    scheduler() {
      dirty = true     // 依赖变化时，标记脏了，但不立即执行
    }
  })

  return {
    get value() {
      if (dirty) {
        value = effect()   // 只在"脏"时才重新计算
        dirty = false
      }
      return value
    }
  }
}
```

关键点：依赖变化时不立即重新计算，而是标记 `dirty = true`，等下次读取时再算。这就是**缓存**机制。

## 一张图总结

```
用户修改数据
     ↓
  Proxy set 拦截
     ↓
  trigger(target, key)
     ↓
找到所有依赖这个 key 的 effects
     ↓
重新执行 effects（更新视图、重新计算 computed 等）
```

反过来：

```
effect 执行（渲染函数、computed getter 等）
     ↓
读取响应式数据
     ↓
  Proxy get 拦截
     ↓
  track(target, key)
     ↓
把当前 effect 记录到依赖集合
```

## 总结

Vue3 响应式的核心思路：**读取时收集依赖，修改时触发更新**。用 Proxy 拦截数据读写，用 WeakMap + Map + Set 存储依赖关系。相比 Vue2，它更彻底、更高效，也能正确处理数组和动态属性。
